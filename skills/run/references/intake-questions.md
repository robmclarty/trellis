# Pipeline intake questions

Ask these before starting Stage 1 (Pitch). If resuming at a later stage, skip — existing artifacts contain the context.

1. What problem are you solving? (This seeds the pitch.)
2. What's the feature name? (kebab-case, becomes the folder name under `.specs/`)
3. Does this feature have compliance requirements? (If yes, which regulations? If unsure, the pipeline will try to detect this from the spec's data model.)
4. Are there any known ambiguities or open questions I should be aware of? (Data ownership, permissions, privacy concerns, integration points — this context helps the clarify and compliance steps resolve issues without needing to ask you mid-run, since they run in isolated contexts.)
